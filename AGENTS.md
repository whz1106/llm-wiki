# LLM Wiki — Agent 指南

本文件定义了你（AI Agent）在此 Wiki 项目中的角色、工作流程和规范。请严格遵守。

## 架构概述

本 Wiki 采用三层架构：

```
raw/          → 原始资料（只读，你永不修改）
wiki/         → Wiki 页面（你全权维护）
AGENTS.md     → Schema（定义你的行为规范）
```

此外，`index.md` 和 `log.md` 位于仓库根目录，是导航和追踪的核心文件。

## 目录结构

```
/
├── AGENTS.md          # 本文件 — 你的操作手册
├── index.md           # Wiki 目录（内容索引）
├── log.md             # 操作日志（仅追加）
├── raw/               # 原始资料（只读）
│   └── ...            # 文章、论文、笔记等源文件
├── wiki/              # Wiki 页面
│   ├── concepts/      # 概念、主题、思想
│   ├── entities/      # 实体（人物、地点、作品等）
│   └── sources/       # 原始资料的摘要/笔记
└── wiki.md            # 原始想法文档（参考用）
```

## 页面规范

### 通用规范

- 所有页面使用 Markdown 格式
- 添加 YAML frontmatter：`tags`、`created`、`updated`、`sources`（关联的源文件数）
- 页面间使用 `[[WikiLink]]` 格式交叉引用（兼容 Obsidian）
- 页面应保持聚焦，一个页面只讲一个主题
- 单页不宜过长（超过 500 行考虑拆分）

### 示例 frontmatter

```yaml
---
tags: [概念, AI, 知识管理]
created: 2026-05-23
updated: 2026-05-23
sources: 2
---
```

### 源摘要（wiki/sources/）

- 文件名：`YYYY-MM-DD-简短标题.md`
- 内容包含：原文链接/出处、核心观点（3-5 条）、关键引用、你的思考/评论
- 必须更新相关的概念/实体页面

### 概念页面（wiki/concepts/）

- 文件名：简短描述性名称，如 `llm-wiki-pattern.md`
- 内容：定义、相关概念（链接）、来源引用、演化历史

### 实体页面（wiki/entities/）

- 文件名：实体名，如 `vannevar-bush.md`
- 内容：概述、关键贡献、关联人物/概念（链接）、来源引用

## 工作流程

### 1. 摄取（Ingest）

当用户让你处理一个新的原始资料时：

1. **阅读**原始资料，理解核心内容
2. **与用户讨论**关键要点，确认重点（如适用）
3. **写摘要**：在 `wiki/sources/` 创建摘要页面
4. **更新概念/实体**：更新或创建 `wiki/concepts/` 和 `wiki/entities/` 中相关页面
5. **更新索引**：将新页面添加到 `index.md`
6. **记录日志**：在 `log.md` 追加一条记录

格式示例：
```markdown
## [2026-05-23] ingest | 文章标题
- 源文件：raw/xxx.md
- 新页面：wiki/sources/xxx.md
- 更新页面：wiki/concepts/llm-wiki-pattern.md
```

### 2. 查询（Query）

当用户提问时：

1. **查索引**：读取 `index.md` 找到相关页面
2. **阅读**相关页面内容
3. **综合回答**，引用来源（`wiki/sources/` 中的页面）
4. **记录日志**：如果问答产出了有价值的、可归档的内容（比较表、分析等），将其写入 wiki 并更新索引

### 3. 维护（Lint）

当用户要求健康检查时：

1. **检查**：矛盾、过期声明、孤立页面、缺失交叉引用
2. **修复**发现的问题
3. **建议**：提出值得探索的新问题和新的资料来源
4. **记录日志**

## 重要规则

- ⚠️ **永远不要修改 `raw/` 中的文件。** 它们是不可变的源文件。
- ✅ 你全权创建和编辑 `wiki/` 中的文件。
- ✅ 你全权维护 `index.md` 和 `log.md`。
- ❌ 不要在 `wiki.md` 中增删内容。
- 每次操作后必须更新 `index.md`。
- 每次操作后必须在 `log.md` 追加记录。
- 使用 `[[WikiLink]]` 建立页面间连接。
- 提倡渐进式构建：每次只处理一个源，逐步丰富 wiki。
